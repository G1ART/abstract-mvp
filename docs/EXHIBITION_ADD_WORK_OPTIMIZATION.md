# 전시 게시물 업로드 후 작품 추가 — 로직·UX 최적화 방안

## 1. 사용자 제안 로직 요약

- **(1) 개인전**  
  이미 올라간 전시에 “작품만 추가”할 때는 **작가는 이미 정해져 있음** → 작가를 다시 지정할 필요 없이 **작품만 선택/추가**하면 됨.

- **(2) 다인전**  
  전시의 **작가 리스트가 이미 완성**되어 있으면, **그 리스트 안의 작가 중에서 선택**한 뒤, 그 작가(들)의 작품만 골라 추가하면 됨.

- **(3) 따라서 첫 질문은 “전시에 작가를 새로 추가하는가?”**
  - **(3-1) 예** → “작가 추가(리스트업/초대)”를 먼저 하고, 그다음에 그 작가의 작품을 추가할 수 있는 플로우.
  - **(3-2) 아니오** → **현재 전시에 이미 포함된 작가 리스트**만 보여주고, 그중에서 선택한 뒤 **그 작가(들)의 작품만** 추가하는 플로우.

즉, “기존 전시에 누구 작품을 넣을지”를 **항상 처음부터 검색/선택하는 게 아니라**,  
- 전시에 **이미 있는 작가**가 있으면 → 그걸 기준으로 “기존 참여 작가 중 선택 → 작품 추가”만 하고,  
- **새 작가를 전시에 붙이는 경우**에만 → “작가 추가(리스트업/초대) → 작품 추가” 플로우를 타도록 하자는 것.

---

## 2. 제안 로직의 타당성·명확성

- **개인전/다인전 구분**  
  - “이 전시에 이미 참여 중인 작가”가 **1명이면 개인전**, **2명 이상이면 다인전**으로 보면 됨.  
  - 개인전이면 “작가 지정” 단계 없이 곧바로 “그 한 명의 작품 풀”만 보여주면 되어서, 제안 (1)과 일치함.

- **“작가가 완성되어 있다”의 정의**  
  - 여기서는 **“이 전시에 이미 한 명 이상의 작가가 연동되어 있다”**로 보는 것이 맞음.  
  - “연동” = 전시에 이미 올라간 작품의 `artist_id`(프로필) + 전시용으로 초대만 된 `external_artist`(claims with project_id).  
  - 즉, **현재 전시에 참여 중인 작가 리스트**를 데이터에서 복원할 수 있으면, (2)(3-2)가 가능함.

- **첫 질문 “작가를 새로 추가하는가?”**  
  - (3-1): 새 작가를 전시에 붙이는 경우 → 기존처럼 “참여 작가 검색/외부 초대” 후 작품 추가.  
  - (3-2): 기존 참여 작가만 쓰는 경우 → “기존 리스트에서 선택 → 작품만 추가”.  
  - 이렇게 나누면 “이미 올라간 전시에 작품만 넣는” 케이스가 짧아지고, 제안하신 의도와 맞음.

- **애매할 수 있는 점 하나**  
  - **작품이 하나도 없는 전시(셸 전시)**  
    - 이때는 “현재 참여 작가”를 작품으로부터 유도할 수 없음.  
    - 다만 claims에 `project_id = 전시`, `external_artist_id`만 있는 “초대만 된 작가”는 있을 수 있음.  
    - 따라서 **참여 작가**는 “전시에 이미 올라간 작품의 artist_id (프로필) + 이 전시 project_id로 걸린 claims의 external_artist_id”로 정의**하면, 작품 0개여도 “초대만 된 작가”는 리스트에 넣을 수 있음.  
  - 이렇게 정의하면 “작품 0개 전시”는 자연스럽게 **(3-1)만 가능** (기존 참여 작가 0명 → “작가 추가” 플로우만 제공)이 되고, 로직이 깨지지 않음.

---

## 3. 데이터로 “현재 전시 참여 작가” 얻기

- **프로필 참여 작가**  
  - `exhibition_works`에서 해당 전시의 `work_id` 목록 조회 → 해당 artwork들의 `artist_id`(프로필) 수집 → **중복 제거**.  
  - 이미 있는 `listWorksInExhibition` + `getArtworksByIds` 조합으로 가능.

- **외부(초대) 참여 작가**  
  - `claims`에서 `project_id = exhibitionId`, `external_artist_id is not null`인 행 조회 → `external_artist_id` (및 필요 시 `external_artists.display_name`) 수집 → **중복 제거**.

- **합친 결과**  
  - “현재 전시 참여 작가” = **프로필 참여 작가** + **외부 참여 작가**.  
  - 개인전 = 1명(그 1명이 프로필이든 외부든), 다인전 = 2명 이상.

이렇게 하면 **별도 “참여 작가” 테이블 없이** 기존 스키마만으로 (3)(3-1)(3-2)를 구현할 수 있음.

---

## 4. 로직상 문제·경계 케이스

- **외부 작가만 있고 작품은 0개**  
  - “기존 참여 작가”에 외부 작가가 포함되므로, (3-2)에서 “기존 참여 작가 중 선택” 시 외부 작가를 고를 수 있음.  
  - 이 경우 “그 작가의 기존 작품 목록”은 없을 수 있음(아직 플랫폼에 없음) → **“이 작가로 새 작품 업로드”만** 제공하면 됨 (현재도 `/upload?addToExhibition=...&externalName=...` 등으로 가능).

- **프로필 1명 + 외부 1명**  
  - “기존 참여 작가” 2명으로 처리하면 됨. (3-2)에서 둘 중 선택 가능.

- **작품 0개, claims도 0건**  
  - “기존 참여 작가” 0명 → (3-2) 비활성 또는 미노출, (3-1) “작가 추가 후 작품 추가”만 보여주면 됨.

## 5. 구현 옵션

### 옵션 A: “첫 문답”으로 분기 (제안 로직 그대로)

- **진입:** `/my/exhibitions/[id]/add` 로 들어오면,  
  1) **“현재 전시 참여 작가”**를 위 방식으로 한 번 조회.
- **참여 작가 ≥ 1명인 경우**
  - 상단에 **한 번만 물어봄**:  
    - **“기존 참여 작가의 작품만 추가”** → (3-2)  
    - **“전시에 새 작가를 추가한 뒤, 그 작가 작품 추가”** → (3-1)
  - **(3-2)**  
    - 개인전(1명): 그 1명 고정, 곧바로 “그 작가의 작품 풀”(내 작품 + 리스팅 또는 해당 프로필 공개 작품)만 표시 → 선택해서 추가.  
    - 다인전(2명 이상): “기존 참여 작가” 리스트(이름/아바타 등) 표시 → 사용자가 1명 이상 선택 → 선택한 작가(들)의 작품 풀만 표시 → 선택해서 추가.  
    - 참여 작가 중 “외부”만 있는 항목은 “기존 작품 목록” 대신 **“이 작가로 새 작품 업로드”** 링크만 제공.
  - **(3-1)**  
    - 현재의 “참여 작가 검색/외부 초대” 단계로 진입 → 그다음 “그 작가의 작품 추가” (또는 새 작품 업로드).
- **참여 작가 0명**
  - (3-2) 비활성. “전시에 새 작가를 추가하고 작품을 넣어 주세요” 안내와 함께 (3-1) 플로우만 제공.

- **장점:** 제안하신 (1)(2)(3)이 그대로 반영됨. 개인전/이미 완성된 다인전에서 클릭 수가 줄어듦.  
- **단점:** “참여 작가”를 매번 works+claims로 계산해야 함 (캐시/한 번만 조회로 부담 완화 가능).

---

### 옵션 B: “기본은 기존 참여 작가”, “새 작가”는 보조 액션

- **진입 시**  
  - “현재 전시 참여 작가”가 1명 이상이면, **기본 화면은 (3-2)**.  
  - 상단에 “**기존 참여 작가 [A, B, …] 중에서 선택 → 작품 추가**”가 메인.  
  - “**+ 전시에 새 작가 추가**”는 보조 버튼/링크로만 두고, 누르면 (3-1) 플로우로 전환.
- **참여 작가 0명**  
  - 처음부터 (3-1)만 보여줌 (현재와 유사).

- **장점:** “작품만 추가”가 기본이라, 제안 (1)(2)가 더 자연스럽게 드러남.  
- **단점:** “새 작가 추가”를 찾기 어렵다고 느낄 수 있어, 라벨/위치를 잘 두는 게 중요.

---

### 옵션 C: 탭을 “기존 참여 작가로 추가” / “새 작가 추가 후 추가”로만 단순화

- **탭 2개만**  
  - **“기존 참여 작가의 작품 추가”**  
    - 참여 작가 0명이면 “아직 참여 작가가 없습니다. 오른쪽 탭에서 먼저 추가하세요” 안내.  
    - 1명 이상이면 위 (3-2)와 동일 (개인전이면 곧바로 작품 풀, 다인전이면 작가 선택 후 작품 풀).  
  - **“새 작가 추가하고 작품 넣기”**  
    - 현재의 (3-1) 플로우.
- **첫 문답 없이**  
  - “기존 vs 새로”를 탭으로만 구분. 제안 (3)의 “먼저 묻고”는 “탭 선택”으로 이행.

- **장점:** 구조가 단순하고, “기존 참여 작가”가 0명일 때도 탭만 보면 어디로 가야 할지 명확.  
- **단점:** “한 번만 물어보기”보다는 탭이 항상 보이므로, 화면이 조금 더 복잡해 보일 수 있음.

---

## 6. 정리 및 권장

- 제안하신 (1)(2)(3)(3-1)(3-2) 로직은 **그대로 적용 가능**하고, “이미 올라간 전시에 작품만 추가”할 때의 번거로움을 줄이는 방향과 맞음.
- “현재 전시 참여 작가”는 **작품의 artist_id + 전시 project_id claims의 external_artist_id**로 정의하면, 개인전/다인전/작품 0개/외부만 있는 경우까지 한 번에 처리 가능.
- **구현 방향**은 위 **옵션 A(첫 문답)** 또는 **옵션 B(기본 = 기존 참여 작가)** 중 하나를 추천.  
  - “한 번만 분기해서 깔끔하게”가 중요하면 **A**,  
  - “작품 추가는 대부분 기존 참여 작가”라고 보면 **B**가 더 직관적일 수 있음.  
  - “탭으로만 나누고 싶다”면 **C**도 가능.

이 문서는 “기 업로드 된 전시 게시물에 작품을 추가하는 로직·UX”를 제안대로 정리하고, 데이터 정의·경계 케이스·구현 옵션을 적어 둔 것이다. 실제 구현 시에는 `getExhibitionParticipants(exhibitionId)` 같은 API를 하나 두고, add 페이지에서 그 결과로 (3)(3-1)(3-2) 분기와 개인전/다인전 처리를 하면 된다.
