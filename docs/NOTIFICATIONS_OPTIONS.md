# 알림(Notifications) UI 옵션 제안

좋아요, 신규 팔로우, 작품 클레임(확정 요청/승인), (추후) 가격 문의 등에 대한 알림을 어디에 어떻게 띄울지 옵션입니다.  
Settings에서 알림 종류별로 on/off 가능하게 하는 것은 추후 적용 예정.

**구현 상태**: 옵션 A(아바타 배지 + 알림 페이지) 적용됨. 마이그레이션 `p0_notifications.sql`, `/notifications` 페이지, 헤더 배지·링크.

---

## 옵션 A: 아바타 옆 배지 (미니멀)

- **위치**: 상단 우측 **아바타 아이콘** 위/옆에 읽지 않은 알림 개수 배지 (빨간 점 또는 숫자).
- **동작**: 클릭 시 드롭다운 또는 전용 패널에 최근 알림 목록 (좋아요, 팔로우, 클레임, 가격 문의 등).
- **장점**: 탑라인이 깔끔함. Instagram/Behance 등에서 많이 쓰는 패턴.
- **단점**: “알림”이 아바타와만 연결되어 있어, 처음 쓰는 사용자는 발견이 다소 늦을 수 있음.

---

## 옵션 B: My Profile 버튼에 배지

- **위치**: **My Profile** 텍스트 링크 옆에 작은 배지 (숫자 또는 점).
- **동작**: 클릭 시 /my로 이동. /my 상단에 “알림” 섹션 또는 탭으로 최근 알림 표시.
- **장점**: “내 것”과 직관적으로 연결. 피드/탐색과 알림이 분리되어 탑라인은 단순 유지.
- **단점**: 알림만 보려고 해도 My 페이지로 들어가야 함.

---

## 옵션 C: 아바타 + My Profile 둘 다 (하이브리드)

- **위치**: 아바타에 배지 + My Profile에도 “알림 N건” 같은 짧은 문구 또는 배지.
- **동작**: 아바타 클릭 시 드롭다운에 “알림” 행 + 최근 N건 미리보기. My Profile 클릭 시 /my에서 전체 알림.
- **장점**: 발견성과 접근성 모두 높음.
- **단점**: 같은 정보가 두 곳에 노출되어 UI가 다소 반복될 수 있음.

---

## 권장

- **가장 미니멀**: **옵션 A** (아바타 배지 + 클릭 시 알림 목록).
- **탑라인 단순 + “내 프로필” 강조**: **옵션 B** (My Profile 배지 → /my에서 알림).
- **발견성 최우선**: **옵션 C**.

---

## 알림 종류 (공통)

- 좋아요 (내 작품/리스팅에 하트)
- 신규 팔로우
- 작품에 대한 클레임 (확정 요청 도착, 승인/거절 결과)
- (추후) **가격 문의**: “Price upon request” 작품에 대한 문의 도착, 작가의 답변 도착 → 알림 + Settings에서 수신 on/off.

---

## 5-1. 가격 문의 (Price inquiry)

- **기능**: 작품이 “Price upon request”일 때, 구매 희망자가 **가격 문의**를 보낼 수 있음. 작가는 문의 목록에서 확인 후 **답변** (비공개 메시지 또는 “문의해 주세요” 링크 등).
- **알림**: 문의 도착 시 작가에게 알림, 답변 도착 시 문의자에게 알림. 위 알림 옵션(A/B/C)과 동일한 채널에 표시.
- **Bidding/negotiation**: 이후 단계로 두고, 1차는 “문의 + 답변”만 구현하는 것을 권장.
