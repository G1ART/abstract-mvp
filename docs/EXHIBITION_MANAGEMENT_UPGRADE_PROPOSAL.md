# 전시 게시물 업로드·관리 고도화 제안

갤러리 입장에서 **전시 게시물 업로드/관리가 가장 핵심적이고 자주 쓰이는 기능**이라는 전제 하에,  
**오프라인 전시 하나 전체를 온라인에서 쉽고 간편하게 매니징**할 수 있는 수준으로 로직·UX를 정리·제안한다.  
예술 도메인에 한정하지 않고, 비슷한 기능을 제공하는 서비스 벤치마킹과 작가 프로필 노출까지 포함한다.

---

## 1. 목표와 원칙

- **심플함**: 한 화면/한 플로우에 역할이 하나에 가깝게. “작품만 추가”와 “새 작가 초대”는 분리되어 보이게.
- **구체적**: 버튼·문구·진입 경로가 명확하고, “어디서 뭘 하는지”가 직관적.
- **고도화**: 다른 앱에서 잘 안 해주는 것 — **전시 단위로 참여 작가·작품·미디어·커버·순서·클레임**을 한곳에서 관리하고, **개인전/다인전·기존 참여 vs 새 작가**를 로직으로 구분하는 수준.

---

## 2. 현재 구조 요약 (기준점)

| 영역 | 현재 구현 | 비고 |
|------|-----------|------|
| **전시 생성** | `/my/exhibitions/new` — 제목, 기간, 상태, 호스트명 입력 후 저장 → `/add`로 이동 | 생성 직후 작품 추가 유도 |
| **전시 수정** | `/my/exhibitions/[id]/edit` — 메타 정보 + 위임 초대 + 삭제(전시만/작품포함) | 편집과 삭제가 한 페이지 |
| **작품 추가** | `/my/exhibitions/[id]/add` — 2단계(참여 작가 → 작품 선택). 참여 작가=풀 필터. 외부 작가 초대·위임 초대 동시 노출 | EXHIBITION_ADD_WORK_UX_AUDIT.md, EXHIBITION_ADD_WORK_OPTIMIZATION.md 참고 |
| **전시 상세(관리)** | `/my/exhibitions/[id]` — 썸네일(커버 3장), 작품(작가별 드래그 순서), 전시전경/부대행사/커스텀 미디어, 작품 제거 | 강력한 편집 기능 |
| **작품 정보 수정** | `/artwork/[id]/edit` — 작품 단위. 전시 컨텍스트에서 “이 전시에서 수정” 진입은 없음 | 전시 내 인라인 수정 없음 |
| **클레임** | 작품 상세에서 요청/승인. 전시에 작품 추가 시 CURATED 클레임 자동 생성. 외부 작가는 add 페이지에서 초대 | 전시 단위 “클레임 대시보드” 없음 |
| **공개 전시** | `/e/[id]` — 제목·기간·호스트/큐레이터·작가별 작품·미디어 버킷 | 로그인 시 “전시 관리” 링크 |
| **작가 프로필** | `listExhibitionsForProfile`: 큐레이션/호스트 + **참여 작가로 참여한 전시** 포함. 탭 “전시”에서 리스트(썸네일+제목+기간+크레딧) | 참여 역할(큐레이터 vs 참여 작가) 구분 표시 없음 |

## 3. 벤치마킹 (참고할 만한 패턴)

- **이벤트/프로젝트 관리**  
  - 한 “이벤트”에 **참가자 추가** vs **기존 참가자만 골라 액션**을 나누는 패턴.  
  - 우리의 “전시에 새 작가 추가?” vs “기존 참여 작가 중 선택” 분기와 동일한 사고.
- **포트폴리오/컬렉션 툴**  
  - “컬렉션” 생성 후 **기존 아이템 추가**를 기본으로 두고, “새로 만들기”는 보조.  
  - 전시도 “기존 참여 작가의 작품 추가”를 기본, “새 작가 초대”는 보조로 두면 갤러리 사용 패턴과 맞음.
- **전시/갤러리 관련 서비스**  
  - Artland, Artsy 등: 전시 페이지는 **타이틀·기간·장소·참여 갤러리/작가·작품** 구조.  
  - 우리는 “참여 작가”를 데이터(작품+claims)에서 복원하고, **개인전이면 작가 선택 없이 작품만**, **다인전이면 참여 작가 중 선택**으로 줄이는 것이 차별점.
- **공유/협업**  
  - “프로젝트에 멤버 초대”와 “이미 있는 멤버만 선택해 작업”을 분리하는 UX.  
  - 전시: “관리자 위임 초대”와 “참여 작가(작품 풀)”를 같은 페이지에 넣지 말고, **작품 추가 플로우**와 **설정/팀**을 분리하면 역할이 분명해짐.

**정리**: “기존 참여자(작가) 기준으로만 액션” + “새 참여자(작가) 추가는 별도 플로우”로 나누고, **작품 추가 페이지는 “작품만 넣기”에 집중**시키는 방향이 업계 패턴과도 맞다.

## 4. 영역별 업그레이드 제안

### 4.0 큐레이터 · 갤러리/호스트 분리 및 상호 초대

- 전시를 **만드는 주체**가 큐레이터일 수도, 갤러리/갤러리스트일 수도 있으므로, **큐레이터**와 **호스트(갤러리·장소)**를 생성/수정 시 명시하고 상호 초대할 수 있어야 한다.
- **갤러리 C**가 전시를 만들고 작품을 올리는 경우: **큐레이터 B**를 “큐레이터”로 지정해 B도 크레딧·관리 권한을 갖게 한다.
- **큐레이터 B**가 전시를 올리는 경우: **갤러리 C**를 “호스트/장소”로 검색·지정해 “어디서 연 전시인지”를 입력한다.
- **갤러리/갤러리스트가 직접 큐레이션**한 경우(한 사람이 큐레이터이자 호스트)는 그대로 지원하며, 정보 입력·권한·프로비넌스는 **업로드/편집 시점에 정리**하고 클레임으로 넘기지 않는다.
- 상세 설계·API·UI·RLS 정리: **`docs/EXHIBITION_CURATOR_HOST_DESIGN.md`** 참고.

### 4.1 전시 게시물 업로드 (생성)

- **현재**: 제목·기간·상태·호스트명 → 저장 후 곧바로 `/add`.
- **제안**  
  - 생성 폼은 유지하되, **“저장 후 할 일”**을 명시: “다음: 참여 작가와 작품을 추가하세요.”  
  - **선택**: “빈 전시로 만들고 나중에 채우기” vs “지금 바로 작품 추가”를 저장 후 한 번만 선택하게 할 수 있음 (기본은 “지금 바로 작품 추가”로 두어도 됨).
- **진입 경로 통일**  
  - `/upload` 탭 “전시 만들기” → `/my/exhibitions/new?from=upload` → 저장 후 `/add` 또는 `/my/exhibitions/[id]`.  
  - “전시 만들기”가 갤러리 핵심이므로 **내 프로필/대시**에서도 동일한 CTA가 눈에 띄게 (이미 “전시 만들기 및 관리” 링크 있음).

### 4.2 전시 수정

- **현재**: edit 페이지에서 메타·위임·삭제가 한꺼번에.
- **제안**  
  - **메타 편집**과 **위임(팀) 관리**를 시각적으로 블록 분리. “전시 정보” / “관리자 초대” / “삭제” 순.  
  - **삭제**는 “전시만 삭제(작품 유지)” vs “전시+작품 삭제”를 이미 하고 있으므로, 문구만 i18n 정리.  
  - **저장 후 이동**: “전시 상세로”가 기본. “작품 추가 페이지로” 옵션은 add 플로우 개선 후 고려.

### 4.3 작품 추가 (핵심)

- **상세 제안은 `EXHIBITION_ADD_WORK_OPTIMIZATION.md`에 정리됨.** 여기서는 요약과 보완만.
- **첫 문답**: “이 전시에 **새 작가를 추가**하시나요?”  
  - **아니오** → **현재 전시 참여 작가**(작품 기반 artist_id + claims의 external_artist_id)만 표시.  
    - 개인전(1명): 작가 선택 없이 곧바로 그 작가의 작품 풀.  
    - 다인전(2명 이상): 참여 작가 중 1명 이상 선택 → 선택한 작가(들)의 작품만 표시 → 선택해서 추가.  
  - **예** → 기존처럼 “참여 작가 검색 + 외부 작가 초대” 후 작품 선택(또는 “이 작가로 새 작품 업로드”).
- **참여 작가 0명**(셸 전시): “새 작가 추가” 플로우만 제공.
- **같은 페이지에서 분리**  
  - **“작품 추가”** 블록과 **“관리자 위임 초대”** 블록을 명확히 구분(섹션/접기).  
  - “작품만 넣으러 왔다”는 유저는 위임 UI에 덜 주의가 가도록.
- **백/네비**  
  - “← 전시 상세로” 등으로 라벨을 **실제 이동 대상**(해당 전시)과 일치시키기 (현재 “Back to My Exhibitions”인데 `/my/exhibitions/[id]`로 가는 문제 해소).

### 4.4 업로드된 작품 정보 수정

- **현재**: 작품 수정은 `/artwork/[id]/edit`에서만 가능. 전시 상세에서 “이 작품 정보 수정” 링크는 없음.
- **제안**  
  - **전시 상세** (`/my/exhibitions/[id]`) 작품 카드에 **“수정”** 링크 추가: `/artwork/[id]/edit?returnTo=/my/exhibitions/[exId]` (또는 `artworkBack`에 전시 URL 저장).  
  - 수정 완료 후 **“전시 상세로 돌아가기”** 옵션 제공 → 갤러리가 “전시 안에서 작품을 손보는” 느낌을 줄 수 있음.  
  - **인라인 편집**(제목/연도 등만 전시 페이지에서 수정)은 2단계로 두고, 1단계는 “작품 상세 수정 + 복귀 경로”만 해도 효과 큼.

### 4.5 클레임 (전시 맥락)

- **현재**:  
  - 전시에 작품 추가 시 `createClaimForExistingArtist(CURATED, projectId)` 자동.  
  - 외부 작가는 add 페이지에서 이름+이메일로 `createExternalArtistAndClaim(projectId)` 후, “이 작가로 새 작품 업로드” 링크.
- **제안**  
  - **전시 단위 “이 전시 관련 클레임” 뷰** (선택): `/my/exhibitions/[id]` 또는 add 페이지에 “이 전시에 대한 클레임/초대 상태” 요약 블록.  
    - 예: “CURATED 클레임 N건”, “외부 작가 초대 보냄 M명” 등.  
  - **작품 상세**에서의 클레임 요청/승인은 유지.  
  - “내 전시에 걸린 클레임 요청(pending)”이 있으면 **내 프로필/전시 목록** 또는 **전시 상세**에서 배지/알림으로 보여주면 갤러리가 놓치지 않음 (이미 `getMyPendingClaimsCount` 등 있음).

### 4.6 공개 전시 페이지 (`/e/[id]`)

- **현재**: 제목·기간·호스트/큐레이터·상태·작가별 작품·미디어 버킷. 관리자만 “전시 관리” 링크.
- **제안**  
  - **썸네일(커버)**: 관리 화면에서 지정한 `cover_image_paths`가 있다면 공개 페이지 상단에 캐로셀/그리드로 노출 (이미 데이터는 있으므로, 공개 쪽에서만 표시 로직 추가).  
  - **작가명 클릭**: 작가별 섹션에서 작가명 → `/u/[username]` 등 프로필로 이동 (이미 프로필 있을 수 있음).  
  - **작품 수/참여 작가 수**를 서브타이틀에 표시 (“N명 작가, M점” 등)하면 전시 규모가 한눈에 들어옴.

### 4.7 작가 프로필에서의 전시 노출

- **현재**: `listExhibitionsForProfile`로 “큐레이터/호스트” + “내 작품이 포함된 전시”를 합쳐서 표시. 탭 “전시”에서 리스트.
- **제안**  
  - **역할 구분**:  
    - “**주최/큐레이션**”(curator/host) vs “**참여**”(작가로 참여)를 라벨/아이콘으로 구분하면, 방문자가 “이 사람이 연 전시”와 “이 사람이 참여한 전시”를 구분 가능.  
  - **카드 정보**:  
    - 썸네일, 제목, 기간, “OO 갤러리 주최” 등 크레딧, **참여 시 “OO전 참여”** 문구.  
  - **정렬**:  
    - “최신순” 유지. 선택적으로 “주최 전시 우선” 필터.  
  - **빈 전시**:  
    - `listExhibitionsForProfile`에서 “작품이 하나도 없는 전시”는 제외하는 옵션을 두면, 작가 프로필에는 “실제로 작품이 올라간 전시”만 노출할 수 있음 (이미 피드용으로 비슷한 필터 있음).

## 6. 정리

- **핵심**: “이미 올라간 전시에 작품만 넣을 때”를 **첫 문답(새 작가 추가?) → 기존 참여 작가만 선택 → 작품만 추가**로 짧게 만드는 것.  
- **데이터**: “현재 전시 참여 작가” = exhibition_works → artworks.artist_id + claims(project_id, external_artist_id). 별도 테이블 없이 구현 가능.  
- **역할 분리**: 작품 추가 페이지는 “작품 추가”에 집중, 위임/설정은 분리. 전시 수정 페이지는 “정보 / 팀 / 삭제” 블록 분리.  
- **작가 프로필**: 전시가 “주최”인지 “참여”인지 구분해 보여주면, 오프라인 전시를 온라인에서 매니징한 결과가 작가에게도 잘 드러난다.

## 7. 세부·섬세한 UX 고민

### 7.1 전시 상태에 따른 문맥

- **planned / live / ended**에 따라 관리 화면 문구를 조금씩 다르게 줄 수 있음.  
  - 예: live일 때 “전시 중 — 작품 추가해도 공개에 바로 반영됩니다”, ended일 때 “종료된 전시 — 이력/아카이브용 수정” 등.  
- 공개 페이지에서는 이미 상태 라벨을 보여주고 있으므로, 관리 쪽에서만 상태를 강조해도 됨.

### 7.2 연속 작업 흐름

- **새 작품 업로드 후**: 현재는 전시 **상세**로 복귀. “한 편 더 추가”하려면 상세 → “작품 추가” 다시 진입.  
  - 선택: 업로드 완료 시 “전시 상세로” / “작품 추가 페이지로 다시” 중 선택하게 하면, 연속으로 여러 작품 올리기 편해짐.
- **작품 추가 페이지**에서 여러 개 “추가” 클릭 후: 이미 한 번에 여러 개 추가 가능. “추가됨” 상태가 곧바로 반영되므로, 추가로 “N개 추가됨 → 전시 보기” 토스트만 있어도 충분할 수 있음.

### 7.3 전시 상세(관리) 내비게이션

- 상단에 “편집” / “작품 추가” / “관리자 초대”가 나란히 있음.  
  - “작품 추가”가 가장 빈번하다면, 버튼 비중(스타일)을 더 크게 두거나, 작품이 0개일 때는 “작품 추가”를 메인 CTA로 두는 식으로 강조 가능.

### 7.4 모바일

- 전시 상세의 **작가별 드래그·작품 순서 변경**이 모바일에서 번거로울 수 있음.  
  - 터치에서 “길게 누르기 → 드래그”가 잘 동작하는지 확인.  
  - 또는 “순서 편집 모드” 진입 시에만 드래그 활성화해 실수 방지.

### 7.5 접근성·i18n

- “전시”, “참여 작가”, “작품 추가”, “주최/큐레이션” 등 핵심 라벨은 모두 i18n 키로 두어 KO/EN 일관되게.  
- 삭제 확인 등 위험 액션은 `role="alert"` 또는 모달로 포커스 잡아 스크린 리더 대응.

### 7.6 공개 링크·공유

- 공개 전시 URL `/e/[id]`를 “공유하기”로 복사하기 쉽게 하면, 갤러리가 SNS·이메일로 전시 링크를 바로 퍼뜨릴 수 있음.  
  - 전시 상세(관리) 또는 공개 페이지 상단에 “링크 복사” 버튼 추가 권장.
